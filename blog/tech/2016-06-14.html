2016-06-14 C语言链表

在开始学习c语言的时候，经常会有这样的问题：统计一个班的学生的个人信息。初学的时候我们就会开一个数组（比如50个）。那么，问题来了，我不知道这个班有多少个人怎么办？这个班的人多于50怎么办？当然，可以开一个很大的数组，大到实际问题中不可能把它涨满。不过，我习惯于挑毛病。如果这个班只有10个人，我们却开了一个100的数组，岂不是太浪费了？！要知道，有时候我们的内存极其宝贵，往往为了节约成本，一块芯片可能只有类似64B（字节）这么小的内存...

然后...链表可以帮大忙了。（有必要说明，链表的用处很多，我这里的甚至不是重点，有时候也不会帮助节约内存，这里是从一个方面来谈论的。）

Content

指针及结构体这里就不引入了，不然又会有很多东西可以讲。

还是需要先贴一份代码，再来慢慢分析。

...

typedef struct INFORMATION
{
	char data[10];
	struct INFORMATION *next;
}INFO;

...
void CreatLinkList()
{
INFO *p,*q;
if((p=(INFO *)malloc(sizeof(INFO)))==NULL)
	{
	//内存申请失败，处理措施
	}
gets(p->data);
while(strcmp(p->data,"quit"))
{
if((q=(INFO *)malloc(sizeof(INFO)))==NULL)
	{
	//内存申请失败，处理措施
	}
gets(q->data);
p->next=q;
p=q;
}
p->next=NULL;
}

到此，一个单向链表就创建完成了。

定义结构体时，成员中有一个指针指向了这个结构体。这未我们把所有结构体串联起来创造了条件。用if((p=(INFO *)malloc(sizeof(INFO)))==NULL) { //内存申请失败，处理措施 }依照结构体的定义方式，向内存申请了一块空间。（未完待续...）
